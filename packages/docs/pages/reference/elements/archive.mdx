# `<archive>`

The `<archive>` element declares which archival or source-file links
should be offered for generated graphics. Place it in the
[`<images>`](./images.mdx) configuration inside [`<docinfo>`](./docinfo.mdx).

Use the `from` attribute to target a particular subtree or image when you
do not want the same archive formats everywhere.

## Syntax

<SyntaxDisplay name="archive" />

## Examples

```ptx-example
<FRAGMENT parents="book docinfo">
    <images>
        <archive>svg pdf</archive>
        <archive from="graphics">tex</archive>
    </images>
</FRAGMENT>
```
